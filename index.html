Declarative

React is a JavaScript library that allows you to create interactive UIs.

It's a declarative library, which makes your code more predictable and easier to debug.

Component-Based

Being component based allows React to encapsulate components and have them be manage their own state.

This means that when your data changes, React updates the components state and that component gets rendered.

Learn Once, Write Everywhere

React allows you to develop new features without having to write new code because it doesn't assume anything about your tech stack.

You can also render React through the server using NodeJS, and also power native applications using React Native

Simple React App

A React component users a render() method, which takes in input data and returns what to display via this.props.

A big advantage of React is the ability to maintain a state. Using this.state, React can maintain internal data and re-render the markup

JSX

JSX may look like a templating language but it's not.

It's a syntax extension of JavaScript, and in React it creates elements.

  • React embraces the fact that rendering logic is coupled with rendering UI logic.
  • Because react uses components, it can neatly couple these types of logic instead of having to combine them through different files.
  • Using JSX with React isn't required, but it helps keep the code more legible visually.
  • Curly braces ({}) allow us to input JavaScript variables into React using JSX.
  • After the JSX is compiled it shows as a JavaScript object.
